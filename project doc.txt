## Challenge Overview

**Track:** Retail / Local Services / Marketplace

**→ Problem Description:**

For cardiac arrest, survival chance drops 7–10% per minute without CPR. Ambulances often need 8–12+ minutes to arrive, while nearby first responders could reach the scene in 0–4 minutes. That time gap is the difference between life and death.

Currently in Switzerland, after calling 144, dispatchers may send a first responder alert through existing apps (Momentum, Save a Life, First Responders Canton Vaud), but they don’t know if a qualified person is actually nearby. An alert pushed directly to medically trained citizens in the immediate vicinity would be faster and more effective — and 144 would also benefit from this system.
**Primary Objective:**

Build an app that triggers a real-time alert to medically trained citizens (paramedics, doctors, nurses, CPR-trained individuals) within a 300–600m radius when a health emergency is reported — via a manual emergency button or an automated smart health tracker signal. The app should share key health data (allergies, blood group, emergency contacts), provide step-by-step guidance, and use proximity-based logic (escalating radius from 300m → 400m → 600m) to ensure the fastest possible first-responder arrival before the ambulance.
## Resources, Tools & Support

**→ Datasets (type, format, access):**

This challenge focuses on the design of the data model and system logic rather than on a predefined dataset. Participants are expected to define and source the required data themselves, using publicly available or synthetic data where appropriate.

The following data types describe the expected structure of the prototype:
- User profile data (structured): name, age, blood group, allergies, emergency contacts, medical certifications
- Geolocation data (real-time GPS coordinates of users)
- Health tracker signals (simulated or via API from wearable devices)
- Reference data on existing Swiss first responder apps (Momentum, Save a Life, First Responders Canton Vaud)

**Available APIs & Services:**

The following types of APIs and services may be used to implement the prototype:
- Geolocation / Maps APIs (e.g., Google Maps, Mapbox) for proximity-based alerting and routing
- Push notification services (e.g., Firebase Cloud Messaging, APNs)
- Health tracker APIs (e.g., Apple HealthKit, Google Health Connect) for automated emergency trigger simulations

Participants are free to choose appropriate services and tools for their implementation.

**→ Supporting Materials:**

The following reference materials may support the development of the prototype:
- CPR / first aid step-by-step guidance protocols (e.g., AHA or Red Cross guidelines)
- Documentation on the Swiss emergency number 144 dispatch workflow
- Reference architecture of existing first responder apps in Switzerland (e.g., Momentum, Save a Life, First Responders Canton Vaud)

These materials serve as contextual guidance and inspiration for system design.

## Deliverables & Judging

**→ Expected Deliverables Final Prototype (2.4.26):**

Participants must submit:
- A working prototype that is accessible to judges remotely (e.g., via deployed link or recorded demo)
- Source code in a public or accessible repository
- A short README explaining:
- System architecture (high level)
- Data model and data sources used
- Matching / ranking logic

**Success Metrics:**

Solutions will be evaluated based on:
- Impact & Relevance – Does the solution meaningfully improve simulated response time and address a real (emergency) coordination need?
- Matching Logic & Technical Quality – Sound prioritisation criteria, clear ranking/routing mechanism, robust system design.
- Innovation & Approach – Creativity and intelligent use of data, algorithms, or AI.
- User Experience & Clarity – Clear interaction flow and understandable decision logic.
- Feasibility & Scalability – Practical implementation and potential to scale beyond a prototype.
